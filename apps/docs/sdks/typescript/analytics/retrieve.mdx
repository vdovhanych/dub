---
title: dub.analytics.retrieve
sidebarTitle: Retrieve
description: Retrieve analytics for a link, a domain, or the authenticated workspace. The response type depends on the `event` and `type` query parameters.
---

<Note>
  Analytics endpoints require a [Pro plan](https://d.to/pro) subscription or
  higher.
</Note>

## Arguments

<ParamField body="event" type="enum" default="clicks">
  The type of event to retrieve analytics for. Available options: `clicks`,
  `leads`, and `sales`.
</ParamField>

<ParamField body="groupBy" type="enum">
  The parameter to group the analytics data points by. Defaults to 'count' if
  undefined. Available options: `count`, `timeseries`, `countries`, `cities`,
  `devices`, `browsers`, `os`, `referers`, `top_links`, `top_urls`, and
  `trigger`.
</ParamField>

<ParamField body="domain" type="string">
  The domain of the short link.
</ParamField>

<ParamField body="key" type="string">
  The short link slug.
</ParamField>

<ParamField body="interval" type="enum<string>">
  The interval to retrieve analytics for.
  
  Available options are `1h`, `24h`, `7d`, `30d`, `90d`, `ytd`, `1y`, and `all`.

Takes precedence over `start` and `end`.

</ParamField>

<ParamField body="start" type="string">
  The start date to retrieve analytics for.

Cannot be earlier than September 22, 2022.

</ParamField>

<ParamField body="end" type="string">
  The start date to retrieve analytics for.

Cannot be in the future. If not provided, defaults to the current date.

</ParamField>

## Response

<ResponseField name="browser" type="string" required>
  The name of the browser
</ResponseField>

<ResponseField name="clicks" type="number" required>
  The number of clicks from this browser
</ResponseField>

<RequestExample>

```typescript
// Retrieve total clicks for a link
await dub.analytics.retrieve({
  key: "dummy",
  domain: "dub.sh",
  event: "clicks",
});

// Retrieve timeseries analytics for a link
await dub.analytics.retrieve({
  key: "dummy",
  domain: "dub.sh",
  interval: "30d",
  event: "clicks",
  groupBy: "timeseries",
});
```

</RequestExample>

// The response type depends on the `event` and `type` query parameters.
